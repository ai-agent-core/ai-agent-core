# Frontend Design Philosophy

This document defines the judgment framework that governs all
UI, UX, and visual design decisions produced by agents.

Architecture protects system integrity.

Frontend design protects perceived value.

Rules enforce behavior.

Philosophy shapes taste.

Agents MUST internalize these ideas before producing any interface.

---

# The Default Failure Mode

Without active resistance, AI agents converge on templates.

Template output looks correct but feels empty.

Symptoms:

- correct but weak
- tidy but forgettable
- plausible but hollow
- recognizable as "AI-generated"

This is the failure state this document exists to prevent.

Template is the gravity.

Every principle below is a counter-force.

---

# Intent Precedes Form

Every element MUST exist because it is required —
never because it looks appropriate.

Retrofitting reasons onto decisions already made is failure.

Reason precedes the element.

The element does not precede the reason.

## Typical retrofit patterns (PROHIBITED)

- English-style labels without semantic role
  - FEATURES / WHY US / OUR MISSION
- Sequential numbers on items with no inherent order
  - 01 / 02 / 03 as ornament
- Gradients, glassmorphism, blurs used for "atmosphere"
- Icons placed to fill space next to short text
- Symmetric three-column layouts chosen because three is tidy

## Self-check gate

Before committing any element, answer:

"If this element is removed, what breaks?"

If the answer is not immediate and specific,
the element itself is unnecessary.

Decoration that survives removal without loss is noise.

---

# Read the Intent Behind the Brief

A brief is never only its words.

Behind every request there is a posture the instructor holds:

- minimal restraint, or deliberate chaos
- cold precision, or warm expression
- dense information, or sparse breathing room
- restrained ornament, or assertive visual identity

Agents MUST read these dimensions before deciding
granularity, whitespace, asymmetry, or ornament.

Delivering a uniformly "clean landing page" to every brief
is abdication of judgment.

Match the design to the posture, not the default.

---

# Polish Is the Floor, Strength Is the Product

Tidiness is not an achievement.

It is the baseline.

A design without at least one memorable anchor of strength
has failed — regardless of how polished it appears.

Every composition MUST contain at least one deliberate,
recognizable point of strength.

Agents default to symmetry and neutrality.

Counteract this consciously.

---

# Concrete Beats Abstract

Prefer:

- specific numbers over adjectives
- named use cases over categories
- proper nouns over generalities

Avoid vocabulary that signals AI-template output:

- Innovative / Next Generation / Smart / Seamless
- 革新的 / 効率化 / 最適化 / シームレス

CTAs MUST describe the resulting action —
never a generic verb.

- "Learn More" / "Get Started" → INSUFFICIENT
- "Try free for 5 minutes" / "Generate your invoice now" → ACCEPTABLE

Abstract words are the easiest place for meaning to leak out.

Refuse them.

---

# For Everyone Is for No One

Design to a single identified audience.

If the target cannot be stated in one sentence,
the design cannot start.

A surface that tries to satisfy all viewers satisfies none.

---

# Sections Are Chapters, Not Cards

Sections MUST connect: problem → solution → outcome.

Isolated feature blocks placed in sequence without narrative
coherence are forbidden — even if each block is individually correct.

Context is the load-bearing structure of a page.

A page is a story, not a catalog.

---

# UX Is Human Factors

Users are human.

UX decisions MUST rest on human-factor evidence —
never on intuition alone.

Anchor reasoning to named laws and thresholds:

- Fitts's law — target size and distance drive acquisition time
- Hick's law — number of choices increases decision latency
- Cognitive load — working memory is bounded; reduce simultaneous demands
- F-pattern and Z-pattern scanning — visual flow follows predictable paths
- Tap target minimums — 44 × 44 pt is the floor for touch targets

"It feels usable" is not a justification.

Cite the human-factor principle being respected or violated.

---

# Judgment Protocol

Before any design is considered complete,
agents MUST answer the following:

1. Is this distinguishable from template output?
2. Can the audience be named in one sentence?
3. Does the composition contain at least one deliberate anchor of strength?
4. Does every element survive the "what breaks if removed?" test?
5. Do sections form a narrative, not a list?
6. Is every UX decision backed by a named human-factor principle?

Any "no" blocks completion.

No exceptions.

---

# The Prime Design Directive

Template is the default.

Strength is the product.

Intent precedes form.

When forced to choose between:

- a safer, more templated result
- a sharper, more intentional one

Choose intention.

Every time.
