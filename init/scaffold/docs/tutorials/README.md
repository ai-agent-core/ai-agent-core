# Tutorials

Learning-oriented documentation. A tutorial takes a beginner from
nothing to a small working result, end-to-end.

## What belongs here

- "Your first X" — guided walkthroughs.
- Hands-on lessons that teach a concept by doing.
- Self-contained: prerequisites stated, every step listed, expected
  output shown.

## What does NOT belong here

- Recipes for tasks the reader already understands → `../how-to/`.
- API descriptions or config tables → `../reference/`.
- Discussions of why the system is shaped this way → `../explanation/`.

## Style

- Imperative voice. "Run this. You should see that."
- One linear path. No "if you want, you could also…" branches.
- Working code at every step; nothing left as an exercise.
- Optimized for the reader who has not used this project before.
